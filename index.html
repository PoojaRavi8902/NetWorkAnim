Networking Notes

1. IP Address and How It Works

An IP address (Internet Protocol address) is a unique number assigned to every device connected to a network. It allows devices to identify and communicate with each other over a network or the Internet.

Just like a house address helps the postal service deliver letters to the correct location, an IP address helps data packets reach the correct device on a network.

Structure of an IPv4 Address

Most networks use IPv4 addresses, which consist of four numbers separated by dots.

Example:

192.168.1.10

Each number is called an octet and can range from 0 to 255. 

192   .   168   .   1   .   10
Octet1   Octet2   Octet3   Octet4

The complete IPv4 address is 32 bits long, with each octet representing 8 bits.

Example in Binary

192.168.1.10

11000000.10101000.00000001.00001010

Computers use the binary version, while humans use the decimal version.

Classes of IPv4 Addresses

Originally, IPv4 addresses were divided into classes based on the first octet.

Class First Octet Range Typical Use
Class A 1 – 126 Very large networks
Class B 128 – 191 Medium sized networks
Class C 192 – 223 Small networks

Examples

10.0.0.1      → Class A
172.16.5.4    → Class B
192.168.1.10  → Class C

Most home and school networks use Class C private addresses, such as: 

192.168.1.x

How IP Addresses Work in Communication

When a device sends data across a network, the data is broken into small units called packets.

Each packet contains:

  • Source IP address
  • Destination IP address
  • Data

Example:

Source: 192.168.1.5
Destination: 192.168.1.10

Routers and switches use the destination IP address to determine where the packet should be sent.

The packet travels across the network until it reaches the device with the matching IP address.

2. DNS (Domain Name System) and How It Works

The Domain Name System (DNS) translates human-friendly website names into IP addresses.

Humans prefer to remember names such as:

google.com
bbc.com
cambridge.org

But computers communicate using IP addresses, for example: 

142.250.190.78

DNS acts like a phone book for the Internet.

Why DNS Is Needed

Imagine trying to remember IP addresses for every website: 

142.250.190.78
151.101.0.81
104.244.42.1

This would be extremely difficult.

DNS allows us to type:

www.google.com

and the system automatically finds the correct IP address.

DNS Resolution Process

When you enter a website address into a browser, several steps occur.

Step 1 – Browser Cache

The browser first checks its local cache to see if it already knows the IP address. If it finds the IP address, it immediately connects to the website.

Step 2 – DNS Request

If the address is not stored locally, the computer sends a request to a DNS server.

This server is usually provided by:

  • Internet Service Provider (ISP)
  • Public DNS services such as Google DNS

Step 3 – DNS Server Lookup

The DNS server searches its records to find the IP address associated with the domain name. 

google.com → 142.250.190.78

Step 4 – DNS Response

The DNS server returns the IP address to the user's computer.

Step 5 – Website Connection

The browser now knows the correct IP address and sends a request to the web server hosting the website.

The server then sends the webpage back to the user's browser.

Summary of the DNS Process

User enters domain name
        ↓
Browser checks cache
        ↓
DNS server lookup
        ↓
IP address returned
        ↓
Browser connects to web server

Why DNS Is Important

DNS makes the Internet easy to use. Without DNS, users would need to remember numeric IP addresses for every website.

DNS provides:

  • Easier website access
  • Faster connections through caching
  • Organized naming of Internet resources
